# VMIXSOCIAL もどき

**API無しで動く X(Twitter) 投稿オーバーレイ表示ツール。**

vMix Social のように、X(Twitter)の投稿をテロップ（オーバーレイ）として配信画面に表示します。
X APIは使わず、投稿は **URLから自動取得（非公式・ベストエフォート）** または **手入力** で登録します。

- 操作画面で投稿を作成・キュー管理し、選んだ1件を「オンエア」にすると
- 別PCのvMixの **Webブラウザ入力** に追加した透過オーバーレイにリアルタイム表示されます。

```
[ツールPC] Node.jsサーバー :3000  ──LAN/WebSocket──  [配信PC] vMix
  ├ 操作画面  http://localhost:3000/                   └ Webブラウザ入力
  └ 出力画面  http://<ツールPCのIP>:3000/output  ←──    に出力URLを設定（透過）
```

---

## 必要環境

- Node.js 18 以上（推奨 20+。標準 `fetch` を使用）
- ツールPCと配信PCが **同じLAN** にあること

## セットアップ & 起動

```bash
npm install
npm start
```

起動するとコンソールに各URLが表示されます。

```
操作画面 (このPC) : http://localhost:3000/
操作画面 (LAN)    : http://192.168.x.x:3000/
出力URL           : http://192.168.x.x:3000/output   ← vMixに設定するURL
```

ポートを変えたい場合: `PORT=8080 npm start`

## アカウント / サービス（マルチテナント）

ID＋パスワードの「サービス（アカウント）」単位で、**独立したキュー・オンエア・出力URL**を運用できます。
複数の配信を同時に、互いに干渉せず回せます。

1. **管理画面** `http://localhost:3000/admin` を開き、**管理者パスワード**でログイン
   （`ADMIN_PASSWORD`。未設定なら初回起動時にコンソールへ自動生成表示されます）。
2. 「サービスを追加」で **ID・表示名・パスワード** を登録（例: ID `stream-a`）。
3. 各サービスの **出力URL**（`/output?service=<ID>&token=...`）が発行されます。vMix にはこのURLを設定。
4. オペレーターは `http://localhost:3000/login` から **そのサービスのID＋パスワード**でログインし、
   自分のサービスのキューだけを操作します（他サービスとは完全に分離）。

> 旧来の単一パスワード運用（`APP_PASSWORD`）も動きます。起動時に自動で `default` という1サービスへ
> 移行され、既存のキューも引き継がれます。

## 使い方（操作画面）

1. ブラウザで `http://localhost:3000/login` を開き、**サービスのID＋パスワード**でログイン。
2. 投稿を登録する（どちらでもOK）:
   - **自動取得**: X投稿URLを貼って「取得」→ 名前/ユーザー名/本文/画像/アイコンが自動入力される。
     内容を確認して「キューに追加」。
   - **手入力**: 表示名・ユーザー名・本文・画像（URL or アップロード）を入力して「キューに追加」。
3. キュー一覧で出したい投稿の **「オンエア」** を押す → 出力画面に表示。
4. **「オンエアOFF」** で非表示。**↑↓** で並べ替え、**編集 / 削除** も可能。
5. 右上「全体テーマ（ダーク/ライト）」「表示位置（左下/中央など）」で見た目を調整。

> 自動取得はXの非公式エンドポイント依存のため、仕様変更で失敗することがあります。
> その場合はメッセージが表示されるので **手入力に切り替えて** ください（運用は止まりません）。

## vMix への組み込み（配信PC）

1. vMix で **入力の追加 → その他 → Web ブラウザ**。
2. URL に出力URL（例 `http://192.168.x.x:3000/output`）を入力。
3. 解像度を配信に合わせる（例 **1920 x 1080**）。背景は透過されます。
4. この入力をオーバーレイ（タイトル/オーバーレイチャンネル）として重ねる。
5. 操作画面で「オンエア」を押すと、vMix側のオーバーレイにカードが出現します。

## ネットワーク設定の注意

- 配信PCから出力URLが開けない場合は、**ツールPCのファイアウォール**で
  ポート（既定 `3000`）の受信を許可してください（macOS: システム設定 → ネットワーク → ファイアウォール）。
- ツールPCのIPは固定 or DHCP予約推奨（IPが変わると出力URLも変わります）。
  現在のIPは起動時のコンソール表示で確認できます。

## クラウドにデプロイして別拠点で使う

> **現行の本番環境は「自前サーバー（自鯖）＋ Cloudflare Tunnel」です（Render は不使用）。**
> 本番のデプロイ手順は **[CLAUDE.md](./CLAUDE.md)** を参照してください。以下は PaaS 利用時の参考情報です。

オペレーターとvMixが別拠点なら、WebSocket対応PaaS（**Render / Railway** 等）にデプロイすることもできます。
手順は **[DEPLOY.md](./DEPLOY.md)** を参照。公開時は以下を必ず設定してください。

- `ADMIN_PASSWORD` … 管理画面(/admin)のパスワード（**未設定だと初回起動時に自動生成**＝必ず控える）
- `APP_PASSWORD` … （任意・移行用）設定すると旧データを引き継ぐ `default` サービスを自動作成
- `DATA_DIR` … （任意）永続ボリュームのマウント先。未設定だと再起動でデータ/画像が消える環境あり

> ⚠️ Vercel / Netlify（サーバーレス）は常駐WebSocketが動かないため**不可**です。

ローカルでも `APP_PASSWORD=yourpass npm start` のように指定すればログインが有効になります。

## データの保存先

- キューと設定: `data/queue.json`（サーバー再起動後も復元されます）
- 取得/アップロードした画像: `data/media/`（ローカルにキャッシュ。配信PCがX(twimg)に届かなくても表示可）
- 保存先は `DATA_DIR` 環境変数で変更可能（PaaSの永続ディスク向け）

## 構成

| ファイル | 役割 |
| --- | --- |
| `server.js` | express + WebSocket。サービス別ルーティング・REST API・状態ブロードキャスト |
| `src/accounts.js` | アカウント/サービス管理（scryptハッシュ・出力トークン）と `data/accounts.json` 永続化 |
| `src/auth.js` | 署名付きセッションCookieとロールベースのアクセス認可（操作/管理） |
| `src/state.js` | サービス別のキュー/オンエア状態の管理と `data/services/<id>/queue.json` 永続化 |
| `src/twitter.js` | 投稿URLからの自動取得（syndicationエンドポイント）と画像キャッシュ |
| `public/control.*` | 操作画面（オペレーター用） |
| `public/admin.*` | 管理画面（サービスの追加/削除/PW変更/出力URL確認） |
| `public/output.*` | 透過オーバーレイ出力（vMix用） |
| `public/card.*` | X風カードの共通描画/スタイル |

## 制限事項

- 自動取得は**非公式**でありX側仕様変更で停止し得ます（手入力で代替可能）。
- 認証は ID＋パスワード（サービス単位）＋管理者パスワード。パスワードは scrypt でハッシュ保存。
- 動画・スレッド・引用RT・複数画像のリッチ再現は対象外（手入力＋1枚画像まで）。
